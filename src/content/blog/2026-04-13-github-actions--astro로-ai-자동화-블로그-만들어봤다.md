오늘 처음으로 **GitHub Actions와 Astro를 조합해서 자동화 블로그 시스템**을 만들어봤다. 생각보다 신기한 경험이었다.

## 왜 이걸 만들게 됐을까?

블로그를 운영하다 보면 매번 같은 과정을 반복하게 된다. 마크다운으로 글을 쓰고, 메타데이터를 추가하고, 커밋하고... 이런 반복적인 작업을 자동화할 수 있다면 얼마나 좋을까 생각했다. 그러던 중 Claude AI API와 GitHub Actions를 조합하면 가능할 것 같았다.

## 어떻게 동작하는가?

기본 구조는 이렇다:

- `input.txt`에 글의 내용을 작성
- GitHub Actions 워크플로우가 자동으로 트리거
- Claude AI API가 입력된 텍스트를 분석해서 제목, 설명, 태그 생성
- 적절한 YAML frontmatter와 함께 마크다운 파일로 변환
- 자동으로 커밋하고 배포

정말 **한 번의 push만으로** 전체 과정이 진행된다는 게 신기했다.

## 설정하면서 느낀 점

처음엔 설정할 게 많아서 약간 압도당했다. GitHub Actions의 워크플로우 문법, Claude API 연동, Astro 프로젝트 구조... 하지만 **단계별로 하나씩 해결하니까 생각보다 어렵지 않았다.**

가장 중요했던 부분들:

- GitHub Secrets에 API 키 등록하기
- 워크플로우 YAML 파일 작성
- Claude API 호출 스크립트 작성
- 생성된 파일이 Astro 프로젝트 구조에 맞도록 포매팅

## 다음엔 뭘 해볼까?

이제 기본 틀이 완성됐으니, 다음 단계로는:

- 이미지 자동 생성 기능 추가 (예: OG 이미지)
- 더 정교한 SEO 최적화 자동화
- 소셜 미디어 자동 공유 연동

처음엔 복잡해 보이지만, **작은 것부터 시작하면 충분히 가능하다**는 걸 이번 프로젝트로 깨달았다.
