## 왜 이사를 결심했나요?

요즘 블로그 운영하면서 느낀 점이 있어요. 제 글들이 흩어져 있으면 관리도 어렵고, 원하는 대로 커스터마이징하기도 제한적이더라고요. 그래서 **깃허브 블로그로 완전히 이전하기로 마음먹었습니다**. 지금 그 과정을 한창 진행 중이에요.

## 현재 진행 상황

### Velog 포스트 자동화 생성

가장 먼저 할 일은 기존 Velog에 있는 모든 포스트를 자동으로 깃허브 블로그 형식으로 변환하는 거예요. 수동으로 하나하나 옮기면 시간이 너무 오래 걸리니까요.

```python
# Velog API를 활용한 자동화 예시
import requests

def fetch_velog_posts(username):
    # Velog의 포스트 데이터를 가져오기
    response = requests.get(f"https://api.velog.io/posts/{username}")
    return response.json()
```

- 기존 마크다운 형식 유지
- 메타데이터(날짜, 태그 등) 자동 추출
- 이미지 링크 업데이트

### 다음 단계

마이그레이션이 완료되면:

- **모든 포스트가 한 곳에 통합**되어 관리가 쉬워질 예정
- 깃허브의 버전 관리 기능도 활용 가능
- 블로그 디자인을 더 자유롭게 커스터마이징 가능

## 마치며

지금까지는 Velog에서 편하게 글을 썼지만, 이제는 **내 블로그를 완전히 내 것으로 만들고 싶다**는 생각이 들었어요. 자동화 스크립트 덕분에 마이그레이션 과정도 훨씬 수월할 것 같습니다.
