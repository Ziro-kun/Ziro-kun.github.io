# 🔎 프로젝트 개요

- Softskill로써 Design Thinking 기법의 이론 교육을 수강하고, 이를 기반으로 한 머신러닝 활용 서비스를 기획하고 발표하는 시간을 가짐

# 🚩 프로젝트 수행과정 요약

- (아이디어 도출) 기획 아이디어를 자유롭게 브레인스토밍 하다가, 얼마 전 본인의 등원길에 장축 트럭이 U턴을 시도하다 6차선 도로를 막아버리면서 교통체증을 유발한 경험을 이야기 하였고, 이를 해결할 수 있는 방법을 머신러닝으로 생각해보자는 결론을 내림(발표자료의 표지로 활용함)

- (문제정의 1) 운전 초보자이며 정보 탐색에 능숙한 조원이 공감 포인트 제시 및 시스템 적용을 위한 환경 정보를 조사함

- (문제정의 2) 운전병 출신이며 드론, 항공기, 자동차 등에 대한 지식이 해박한 조원이 문제 정의 및 서비스 구상의 와이어프레임을 제시함

- (기술 탐색) 본인은 아이디어를 구현하기 위해 필요한 데이터와 적용할 기술을 탐색하고 이를 서비스의 어떤 부분에 적용할지 매핑하는 과정을 수행함, 이 과정에서 SI 프로젝트 경험을 통해 알게 된 지식을 최대한 활용함

- (서비스 기획) 각자의 아이디어 및 자료를 종합하여 prototype의 개요를 구성하고 어떤 방식으로 제품화 할 지를 논의함

- (아키텍처 설계) 논의한 prototype 결과물을 바탕으로 본인이 간단한 아키텍처를 작성함

# 🪄 활용된 기술(모델)

- 해당 환경에서 차량의 U턴 가능성 여부 산출 : 기하학적 모델(Geometric Model)

- ML 기반 개인화 U턴 성공 확률 모델 : Gradient Boosting / Random Forest / XGBoost/ LightGBM등의 분류 모델을 통해 평소 운전습관 + 장축 + 도로정보를 기반으로 실제 성공 확률을 예측

- 선택적으로 적용할 Computer Vision & Road Scene Classification 모델 : CNN 혹은 ViT기반의 Scene Understanding으로 도로 폭, 표지판, 경사도, 신호 구조를 자동적으로 식별

# ⚠️ 프로젝트 과정 중 발생한 문제-해결 과정

- 약 2~30분 내에 결과물을 도출해야 하는 초 미니 사이드 프로젝트여서 어느 정도까지 제시해야 하는지 정하기 어려웠음

- 팀원들과 논의하여 ‘단기간 내 실제 구현 가능한’모델을 구상하는데 포커싱 하자고consensus를 맞춤

# 📊 회고 혹은 반성

- 기술적으로 ‘단기간 내 실제 구현 가능한’ 모델이라는 부분에 집중하다 보니, 강사님께도 이러한 노력이 잘 전달되어 긍정적 피드백을 받았음

- 별도의 비즈니스 모델이 보이지 않으며 수익성이 있는 제품일지에 대해서는 확신이 들지 않는다는 부정적 피드백도 주셨음

- 강사님의 피드백은 본인이 창업 동아리 학생들의 프로젝트를 피드백할 때 늘 하던 말이기도 한데, 막상 나 자신은 짧은 시간에 너무 ‘구현 가능성’만을 추구하다 보니 비즈니스 모델에 대해 구체적인 답변을 내놓지 못한 것이 아쉬웠음
