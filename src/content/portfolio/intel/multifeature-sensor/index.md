# 🔎 프로젝트 개요

- **UCI Human Activity Recognition(HAR) 데이터셋**을 활용하여, 고차원 센서 데이터 기반 신체 활동을 **비지도 학습·차원축소·시각화 관점에서 분석**하는 캡스톤 프로젝트를 수행함

- 행동 인식 문제에서 **차원축소가 계산 효율·실시간성·분류 성능에 미치는 영향**을 실험적으로 검증하는 것을 목표로 함

# 🚩 프로젝트 수행과정 요약

- (문제 정의) 4W 프레임을 활용하여 웨어러블·스마트폰 기반 행동 인식 서비스의 활용 맥락과 문제 범위를 정의함

- (데이터 이해) 561차원 센서 feature로 구성된 HAR 데이터 구조를 파악하고, 별도 결측치가 없어 원본 구조를 유지한 채 분석을 진행함

- (비지도 학습) Activity 라벨을 제거한 상태에서 Elbow 기법을 적용하여 적정 클러스터 수를 **k=5 또는 k=6**으로 도출함

- (Clustering 분석) k-means 수행 후, cluster별 통계·시각화를 통해 **k=6은 일부 정적 행동에서 과다 분할**이 발생함을 확인함

- (차원축소 실험) PCA를 적용하여 차원 수에 따른 분류 성능·설명력 변화를 비교하고, t-SNE를 활용해 군집 구조를 시각적으로 검증함

- (평가 비교)
    
    - k-means → PCA → 분류
    
    - PCA 단독 → 분류
        
        두 파이프라인의 성능 및 연산 시간 차이를 비교 분석함
        
    

# 🪄 활용된 기술(모델)

- **비지도 학습**
    
    - K-means clustering, Elbow method
    
    - Silhouette, Homogeneity, Completeness, ARI, AMI 지표
    

- **차원축소 및 시각화**
    
    - PCA (다차원 실험), t-SNE (2D·3D 시각화)
    

- **분류 모델**
    
    - RandomForestClassifier
    

- **전처리 및 분석**
    
    - StandardScaler, pandas, numpy
    
    - matplotlib, seaborn 기반 시각화
    

# ⚠️ 프로젝트 과정 중 발생한 문제-해결 과정

- Elbow 기준으로는 k=6이 합리적으로 보였으나, 실제 cluster 특성을 해석한 결과 **정적 행동(LAYING, SITTING, STANDING)에서 의미 중복**이 발생함
    
    → 수치적 지표뿐 아니라 **행동 의미 기반 해석의 필요성**을 인식함
    

- PCA 이후 2차원에서도 분류 성능이 과도하게 높게 나오는 현상이 발생함
    
    → k-means로 이미 기계적 분리가 이루어진 데이터를 기반으로 차원축소를 수행했기 때문임을 확인함
    

- 개인별 예습 코드와 팀 실습 결과가 달라 혼란이 있었음
    
    → **k-means 수행 여부, y 설정 기준(activity vs cluster)** 차이가 원인이었음
    

- 실험 조건을 완전히 통일하지 않은 상태에서 병렬 실험을 진행한 점이 결과 취합을 어렵게 함

# 📊 회고 혹은 반성

- 비지도 학습 결과가 반드시 정답은 아니며, **연구자가 정의한 클래스 체계와 데이터 수집 의도**를 함께 고려해야 함을 깨달음

- 차원축소의 목적이 단순 시각화가 아니라, **모바일·웨어러블 환경에서의 실시간성 확보**라는 점이 인상 깊었음

- 실험 과정에서 모든 중간 결과를 시각화하고 수치로 확인하는 습관의 중요성을 체감함

- 각자 실험을 자유롭게 진행하더라도, **핵심 조건은 반드시 통일한 뒤 비교**해야 한다는 교훈을 얻었음

- 다른 팀의 발표를 보며, 스토리텔링·문제 정당성·역할 기반 협업 측면에서 스스로가 다소 독단적이었다는 점을 깨달았음
    
    → 이후 프로젝트에서는 **과정 공유·중간 점검·설명 가능한 실험 설계**를 우선할 예정임
