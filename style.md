이 가이드는 기존 포스트들을 분석해 도출한 실제 글쓰기 스타일이다.

## 말투 & 어조

### 존댓말 기반, 단 대화체로
- 기본은 **합쇼체/해요체** 혼용: `~했습니다`, `~입니다`, `~더라고요`, `~거든요`, `~잖아요`
- 딱딱한 문어체는 지양. 실제로 이야기하듯 풀어서 쓴다.
- 1인칭: **저, 제** 사용

**좋은 예시:**
> 집에 돌아와 클로드로 마이그레이션을 시작했지만, 리눅스 환경변수와 JSON 구조는 호락호락하지 않았습니다.

> 사실 걔가 하는 건 별 거 없습니다. 일은 우리 LLM API 들이 다 해주죠.

### 유머와 자조적 표현
- 삽질, 멘탈 터짐, 고생담은 솔직하게, 유머러스하게 표현
- `(?)` — 확신 없거나 농담 섞일 때
- `~~취소선~~` — 자조적 유머, 결과론적 반성

**예시:**
> ~~_우리 애기 예쁘죠?_~~

> ~~(안함)~~, ~~(자매품 --force)~~

## 인용구 (`>`) 활용법

핵심 생각, 감탄사, 혼잣말, 유머러스한 상황 묘사에 적극 활용:

```markdown
> Ctrl + Shift + c, v 는 누구 머릿속에서 나온 단축키 입니까?

> "이거 나도 해볼 수 있지 않을까?"

> 왕사남에서 추종자들이 강 건너 유배지의 단종을 보는 심정이 이랬을까요?
```

- 인용구 안에서도 `~~취소선~~` 사용 가능
- 질문 형태로 독자를 끌어들이는 데 자주 사용

## 이미지 삽입법

### 이미지 업로드 & 게시 정책 (2026-04-14부터 적용)

**원칙: 이미지는 먼저 깃허브에 커밋&푸시한 후, HTML img 태그로 참조**

#### Step 1: 이미지를 GitHub에 업로드
```bash
# 1. src/image/ 폴더에 이미지 저장
# 2. 파일명: 자유 (공백, 한글 OK)

git add src/image/파일명.png
git commit -m "add: 블로그 이미지 파일명"
git push
```

#### Step 2: 블로그 글에서 HTML img 태그로 삽입
```html
<img src="https://raw.githubusercontent.com/Ziro-kun/Ziro-kun.github.io/main/src/image/파일명.png" alt="이미지설명" style="width:100%;max-width:800px;" />
~~_짧은 코멘트_~~
```

### 규칙
- ✅ **반드시 HTML `<img>` 태그 사용** (마크다운 `![]()`는 파서 문제로 사용 금지)
- ✅ GitHub raw URL 형식: `https://raw.githubusercontent.com/Ziro-kun/Ziro-kun.github.io/main/src/image/파일명`
- ✅ 파일명에 공백/한글 있어도 OK (GitHub가 자동 처리)
- ✅ `style="width:100%;max-width:800px;"`로 반응형 사이즈 유지
- ✅ 이미지 캡션: 바로 아래에 `~~_짧은 설명_~~` 형식 (회색 작은 이탤릭으로 자동 스타일링)

### 예시
```html
<img src="https://raw.githubusercontent.com/Ziro-kun/Ziro-kun.github.io/main/src/image/스크린샷 2026-04-14 19-40-57.png" alt="에러 메시지" style="width:100%;max-width:800px;" />
~~_이 에러를 보고 깨달았어요_~~
```

## 글 구조

### 제목 체계
- 대제목: `# 숫자. 제목` (예: `# 0. 서론`, `# 1. 사건의 발단`)
- 중제목: `## 숫자. 제목`
- 소제목: `### 소제목` (Stage 1, Step 1 등)
- 번호 없는 소제목도 사용: `### 👥 세 친구 소개`

### 글의 흐름
1. **도입** — 글을 쓰게 된 계기, 배경 상황을 개인 경험으로 풀어낸다
2. **본론** — 단계별로 섹션을 나눠 상황 전개
3. **결말** — 배운 점, 교훈, 독자에게 전하는 말

### 강조 표현
- `**굵게**` — 핵심 키워드, 중요한 결론
- `→` — 인과관계, 다음 단계로 이어질 때
- 목록: `*` 또는 `-` 사용
- 표: 비교, 스펙 정리에 적극 활용

## 구성 요소별 팁

### 기술 내용 서술 시
- 코드는 반드시 코드블럭으로
- 명령어는 인라인 코드(`` `명령어` ``)로
- 어려운 개념은 "~라고 보시면 됩니다", "~인 셈이죠" 같은 표현으로 부드럽게

### 개인 경험담 서술 시
- 시간 흐름 중심으로 서술 (자정까지, 이틀 동안 등)
- 감정 표현 솔직하게: "멘탈이 무너지기 시작했습니다", "너무나도 간절했습니다"
- 실패와 삽질도 숨기지 않고 그대로 서술

### 마무리 패턴
- 독자에게 직접 말 걸기: "채신기술을 두려워 하지 마십쇼. 그들은 우리의 좋은 친구입니다."
- 핵심 교훈을 bullet로 정리
- 짧고 임팩트 있게 끝내기

## input.txt 포맷

### 일반적인 글쓰기
```
글 내용...
```

### 썸네일(Hero Image) 설정

글 시작 부분에 hero-image 메타섹션을 추가하세요:

```
[hero-image: my-hero.png]

글 내용...
```

**규칙:**
- 이미지는 `src/assets/blog/` 폴더에 저장
- 파일명은 정확하게 지정
- Claude가 frontmatter에 `heroImage: ../../../assets/blog/{파일명}` 자동 추가
- Astro가 자동으로 이미지 최적화

### 본문 이미지 포함해서 쓰기

글 시작 부분에 이미지 메타섹션을 추가하세요:

```
[images: content-img-1.png, content-img-2.png]

글 내용...
```

**규칙:**
- 이미지는 쉼표로 구분
- 파일명은 `src/image/` 폴더에 실제로 있는 파일명과 정확하게 일치해야 함
- Claude가 이 목록을 보고 마크다운에 자동 삽입
- 메타섹션은 자동으로 제거되고, 글 내용만 처리됨

### 썸네일 + 본문 이미지 함께 사용

```
[hero-image: my-hero.png]
[images: content-img-1.png, content-img-2.png]

오늘 새로운 기능을 완성했다.

...
```
