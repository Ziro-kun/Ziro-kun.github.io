# 블로그 글쓰기 스타일 가이드

## 전반적인 톤
- 친근하고 대화체로 작성
- 너무 딱딱하거나 학술적인 표현 지양
- 독자가 이미 기초는 알고 있다고 가정하고 설명

## 구조
- 제목은 핵심 내용을 담되 흥미를 유발할 것
- 도입부에서 "왜 이걸 쓰게 됐는지" 간단히 언급
- 소제목으로 내용을 명확히 구분
- 마무리에 배운 점 또는 다음 단계 간단히 정리

## 마크다운 규칙
- 코드는 반드시 코드블럭으로 표시
- 강조할 내용은 **굵게**
- 목록은 `-` 사용

## YAML frontmatter 형식 (반드시 포함)
```
---
title: "제목"
description: "한 줄 요약"
pubDate: YYYY-MM-DD
tags: ["태그1", "태그2"]
---
```

## 언어
- 한국어로 작성
- 기술 용어는 영어 원문 병기 가능 (예: 상태관리(State Management))
